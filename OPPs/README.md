# OOPs Module in Java

This module contains basic examples of **Object-Oriented Programming System (OOPs)** concepts in Java.

OOPs is a programming approach where we write code using **classes** and **objects**. It helps us organize code in a clean, reusable, and easy-to-understand way.

---

## Topics Covered

This module includes examples of the main OOPs concepts:

1. Object and Class
2. Encapsulation
3. Inheritance
4. Polymorphism
5. Abstraction

## 1. Object and Class

A **class** is a blueprint or design.

An **object** is a real instance created from a class.

### Example ConceptHere:

## 2. Encapsulation

**Encapsulation** means hiding data and allowing access through methods.

In Java, we usually make variables `private` and access them using getter and setter methods.

### Example Concept### Benefits

- Protects data
- Improves security
- Gives controlled access
- Makes code easier to maintain

### Covered InThis section explains:

- Private variables
- Getter methods
- Constructor initialization
- Access control

---

## 3. Inheritance

**Inheritance** means one class can use the properties and methods of another class.

The class that gives properties is called the **parent class** or **superclass**.

The class that receives properties is called the **child class** or **subclass**.

### Example ConceptHere:

- `Vehicle` is the parent class.
- `Car` is the child class.
- `Car` can use properties and methods of `Vehicle`.

### Benefits

- Code reusability
- Less duplicate code
- Better structure

### Covered InThis section explains:

- Parent class
- Child class
- `extends` keyword
- Constructor calling using `super`
- Method reuse

---

## 4. Polymorphism

**Polymorphism** means one thing can have many forms.

In Java, polymorphism is mainly of two types:

1. Compile-time polymorphism
2. Runtime polymorphism

---

### Compile-Time Polymorphism

Compile-time polymorphism is achieved using **method overloading**.

Method overloading means having multiple methods with the same name but different parameters.

### Example Concept---

### Runtime Polymorphism

Runtime polymorphism is achieved using **method overriding**.

Method overriding means a child class provides its own implementation of a parent class method.

### Example Concept### Covered InThis section explains:

- Method overloading
- Method overriding
- `@Override` annotation
- Same method name with different behavior

## 5. Abstraction

**Abstraction** means hiding internal details and showing only important features.

In Java, abstraction can be achieved using:

- Abstract classes
- Interfaces

### Example ConceptA class that extends an abstract class must provide implementation for abstract methods.

### Benefits

- Hides unnecessary details
- Improves code structure
- Helps focus on important behavior

### Covered InThis section explains:

- Abstract class
- Abstract methods
- Method implementation in child class

## Important OOPs Keywords Used

| Keyword | Meaning |
|---|---|
| `class` | Used to create a class |
| `object` | Instance of a class |
| `new` | Used to create an object |
| `this` | Refers to the current object |
| `private` | Accessible only inside the class |
| `public` | Accessible from anywhere |
| `extends` | Used for inheritance |
| `super` | Used to call parent class constructor or methods |
| `abstract` | Used to create abstract classes or methods |
| `@Override` | Shows that a method is overridden |
